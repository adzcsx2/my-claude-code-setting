# Quality Rubric

## Purpose

Define how a generated Skill draft is reviewed before promotion.

This rubric evaluates the package itself, not just whether the underlying idea was
worth capturing.

---

## Score Dimensions

Use 1-5 per item:

1. **Symptom specificity**
   - can the trigger conditions uniquely point to this package?
2. **Root cause depth**
   - does it explain mechanism, not just surface observations?
3. **Wrong-path value**
   - does it preserve failed approaches or negative lessons where relevant?
4. **Execution clarity**
   - can someone act on the package after reading it?
5. **Transferable insight**
   - does the package preserve the principle, not just the incident?
6. **Description/body alignment**
   - do frontmatter and body describe the same problem correctly?

---

## Review Outcomes

### Strong draft

- coherent
- specific
- actionable
- internally aligned

### Weak draft

- vague symptoms
- shallow root cause
- no useful wrong-path preservation
- unclear execution path
- mismatched description and body

---

## Minimum Rule

Low-quality drafts should not be promoted directly.

Route them to:

- `inbox` for revision, or
- `quarantine` when confidence is meaningfully low

---

## Notes

Backfill packages should generally be scored more conservatively than realtime
packages because evidence quality is often weaker.
